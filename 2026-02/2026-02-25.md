## 1. rebane2001/x86CSS

> **"用 CSS 造 CPU：当样式表变成汇编器"*

- **语言**: HTML
- **创建时间**: 2026-02-20
- **Forks**: 7
- **许可证**: MIT License

- **项目地址**: https://github.com/rebane2001/x86CSS

### 核心功能

**x86CSS 是一个纯粹用 CSS 实现的 x86 模拟器**，能够在浏览器中运行简单的 x86 程序。

**技术特点**：
- **纯 CSS 实现** — 不使用 JavaScript，完全依赖 CSS 选择器和计数器
- **x86 指令集支持** — 支持基本的 x86 指令（MOV, ADD, SUB, JMP 等）
- **执行机制** — 利用 CSS 的 `:has()` 伪类和兄弟选择器模拟 CPU 执行
- **可视化执行** — 在浏览器中直接看到程序执行过程

**实现原理**：
- 使用 CSS 计数器模拟寄存器
- 通过 `:checked` 状态模拟内存位
- 利用 CSS 变换（transform）和过渡（transition）模拟指令执行周期

### 项目亮点

1. **极限技术探索** — 证明 CSS 的图灵完备性，突破样式表的设计边界
2. **教育价值** — 帮助开发者理解计算机体系结构和 CSS 的深层机制
3. **创意编程** — 将技术约束变成创新动力
4. **传播效应** — 极强的视觉和概念冲击力，易于传播

**这不是一个实用的项目，但这是一个"让人惊掉下巴"的项目。当你意识到 4000 行 CSS 代码能运行 x86 程序时，你会重新审视对 CSS 的认知。**

**2. Web 平台的"操作系统化"趋势**

lifo 反映了 Web 技术的演进方向：

**历史脉络**：
- 静态网页 → 动态网页 → Web 应用 → Web 操作系统

**驱动因素**：
- **PWA 技术** — 离线能力、安装能力
- **WebAssembly** — 接近原生的性能
- **现代浏览器** — 文件系统 API、通知 API 等

**为什么趋势重要**：
- **降低门槛** — 用户无需安装应用，浏览器即平台
- **跨平台统一** — 一次开发，多端运行
- **云端化** — 应用天然运行在云端

**洞察**：Web 正从一个"文档展示平台"进化为一个"应用平台"。lifo 只是这个趋势的早期实验，未来会有更多"浏览器原生操作系统"出现。
