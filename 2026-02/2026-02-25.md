## 1. cloudflare/vinext

> **"Next.js API 在任意平台上运行：Vite 插件让 Next.js 应用脱离 Vercel"**

- **语言**: TypeScript
- **创建时间**: 2026-02-24
- **Forks**: 10
- **许可证**: MIT License
- **ID**: 1165371200
- **项目地址**: https://github.com/cloudflare/vinext

### 核心功能

**vinext 是一个 Vite 插件**，它重新实现了 Next.js 的 API 表面，使得 Next.js 应用可以部署到任何平台。

**核心功能**：
- **API 表面兼容** — 重新实现 Next.js 的核心 API
- **Vite 生态集成** — 作为 Vite 插件运行
- **任意平台部署** — 不限于 Vercel，支持 Cloudflare Workers、Netlify 等平台
- **开发体验保留** — 保持 Next.js 的开发模式和工具链

**目标场景**：
- 将 Next.js 应用迁移到 Vercel 以外的平台
- 利用 Vite 的快速构建能力
- 降低部署成本，选择更便宜的托管服务
- 在边缘平台上运行 Next.js 应用

### 项目背景

**Next.js 依赖问题**：
- Next.js 通常与 Vercel 平台深度绑定
- 部署到其他平台需要大量适配工作
- 开发者希望有更多部署选择

**vinext 的解决方案**：
- 用 Vite 插件方式重新实现 Next.js API
- 解耦 Next.js 与特定平台的绑定
- 保留 Next.js 的核心功能和开发体验
