## 1. peteromallet/dataclaw

> **"Anthropic 拔掉了梯子，DataClaw 把梯子扔回去"*

- **语言**: Python
- **创建时间**: 2026-02-24
- **Forks**: 203

- **项目地址**: https://github.com/peteromallet/dataclaw

### 核心功能

**DataClaw 是一个行为艺术项目，将私人 AI 对话转化为可分享的公共数据集**。Anthropic 用开源信息训练模型，然后限制数据共享 —— DataClaw 让用户夺回数据所有权。

**核心特性**：
- **多源支持** — Claude Code, Codex, Gemini CLI, OpenCode, OpenClaw
- **隐私保护** — 路径匿名化、用户名哈希、密钥检测、邮件编辑
- **数据结构化** — 会话、消息、工具调用、Token 使用统计
- **Hugging Face 集成** — 一键发布为开源数据集

**隐私机制**：
1. 路径匿名化 — 文件路径剥离为项目相对路径
2. 用户名哈希 — 稳定的用户标识替换真实用户名
3. 密钥检测 — 正则表达式捕获 JWT 令牌、API 密钥
4. 熵分析 — 高熵字符串标记为潜在密钥
5. 邮件编辑 — 个人邮件地址移除
6. 自定义编辑 — 额外配置要编辑的字符串

### 数据模式

每个会话一行 JSON：

```json
{
  "session_id": "abc-123",
  "project": "my-project",
  "model": "claude-opus-4-6",
  "git_branch": "main",
  "start_time": "2025-06-15T10:00:00+00:00",
  "end_time": "2025-06-15T10:30:00+00:00",
  "messages": [...],
  "stats": {...}
}
```

### 工作流程

```
1. 安装: pip install dataclaw
2. 配置源: dataclaw config --source all
3. 列出项目: dataclaw list --source all
4. 配置排除: dataclaw config --exclude "personal-stuff"
5. 本地导出: dataclaw export --no-push
6. 审查确认: dataclaw confirm --full-name "NAME"
7. 发布: dataclaw export --publish-attestation "..."
```

### 项目亮点

1. **对抗数据垄断** — AI 公司"拔梯子"，用户"扔回去"
2. **分布式数据集** — 每个导出标记 `dataclaw`，形成人类-AI 编程协作的分布式数据集
3. **隐私优先** — 多层隐私保护机制，确保安全共享
4. **开源即用** — 一条命令发布到 Hugging Face

**当 AI 公司开始限制数据共享时，社区会找到夺回数据主权的方式。DataClaw 代表了"数据民主化"的新浪潮。**

### 核心功能

**Polymarket CLI 是官方 Rust 命令行工具**，提供完整的预测市场操作能力 —— 浏览市场、下单、管理仓位、链上交互，可作为终端工具或脚本/代理的 JSON API。

**核心特性**：
- **无钱包浏览** — 无需钱包即可浏览市场、查看订单簿
- **交易功能** — 下限价单、市价单、批量下单、取消订单
- **数据查询** — 价格历史、订单簿、最新交易、持仓数据
- **JSON 输出** — 所有命令支持 JSON 输出，便于脚本集成
- **多种签名类型** — Proxy (默认), EOA, Gnosis Safe

**支持的提供商**：
- Homebrew (macOS/Linux)
- Shell script 安装
- 从源码构建 (Cargo)

### 命令分类

**市场浏览** (无钱包)：
```bash
polymarket markets list --limit 10
polymarket markets search "election"
polymarket markets get will-trump-win-the-2024-election
polymarket -o json markets list --limit 3
```

**事件管理**：
```bash
polymarket events list --tag politics --active true
polymarket events get 500
```

**订单簿与价格** (无钱包)：
```bash
polymarket clob price TOKEN_ID --side buy
polymarket clob midpoint TOKEN_ID
polymarket clob spread TOKEN_ID
polymarket clob book TOKEN_ID
polymarket clob batch-prices "TOKEN1,TOKEN2" --side buy
```

**交易** (需钱包)：
```bash
# 下限价单 (买入 10 股，$0.50)
polymarket clob create-order --token TOKEN_ID --side buy --price 0.50 --size 10

# 下市价单 (买入 $5)
polymarket clob market-order --token TOKEN_ID --side buy --amount 5

# 批量下单
polymarket clob post-orders --tokens "TOKEN1,TOKEN2" --side buy --prices "0.40,0.60" --sizes "10,10"

# 取消订单
polymarket clob cancel ORDER_ID
polymarket clob cancel-all
```

**链上操作**：
```bash
# CTF 操作
polymarket ctf split --condition 0xCONDITION... --amount 10
polymarket ctf merge --condition 0xCONDITION... --amount 10
polymarket ctf redeem --condition 0xCONDITION...

# 合约授权
polymarket approve check
polymarket approve set  # 发送 6 笔链上交易
```

### 配置与钱包

钱包配置方式（优先级）：
1. CLI flag: `--private-key 0xabc...`
2. 环境变量: `POLYMARKET_PRIVATE_KEY=0xabc...`
3. 配置文件: `~/.config/polymarket/config.json`

创建新钱包：
```bash
polymarket wallet create  # 生成随机密钥并保存到配置
polymarket wallet import 0xabc123...
polymarket wallet show
```

### 项目亮点

1. **官方出品** — Polymarket 官方维护，质量保证
2. **Rust 实现** — 高性能、内存安全
3. **双模式输出** — 人类可读的表格 + 机器可读的 JSON
4. **Agent 友好** — JSON API 模式专为脚本和 Agent 设计

**预测市场正在从 Web 应用走向 CLI。Polymarket CLI 的推出，标志着"终端交易"成为可能 —— 就像 DeFi 从网页到 CLI 的演进。**

## 3. mgechev/skills-best-practices

> **"Agent 技能开发不是艺术，而是工程"*

- **创建时间**: 2026-02-24
- **Forks**: 58
- **许可证**: N/A
- **ID**: R_kgDORYAPPQ
- **项目地址**: https://github.com/mgechev/skills-best-practices

### 核心功能

**skills-best-practices 是 Agent 技能开发的最佳实践指南**，解释如何编写专业级技能、用 LLM 验证它们、并保持精简的上下文窗口。

**技能目录结构**：
```
skill-name/
├── SKILL.md              # 必需: 元数据 + 核心指令 (<500 行)
├── scripts/              # 可执行代码 (Python/Bash) - 微型 CLI
├── references/           # 补充上下文 (schemas、cheatsheets)
└── assets/               # 输出中使用的模板或静态文件
```

### 核心原则

**优化 Frontmatter 以提升可发现性**：
- **严格命名** — name 字段必须 1-64 字符，只包含小写字母、数字和连字符（不能连续），**必须与父目录名完全匹配**
- **触发优化的描述** — 最多 1,024 字符，这是 Agent 路由的唯一元数据，用第三人称描述能力，包含"负向触发器"

**渐进式披露和资源管理**：
- **保持 SKILL.md 精简** — 主文件限制在 < 500 行，用于导航和主要流程
- **使用扁平子目录** — 将大量上下文移到标准文件夹，文件严格保持一级深度
- **按需加载 (JiT)** — 显式指示 Agent 何时读取文件
- **显式路径** — 始终使用带有正斜杠 `/` 的相对路径，无论操作系统

**具体的程序化指令**：
- **使用分步编号** — 定义工作流为严格的时间顺序序列
- **提供具体模板** — Agent 模式匹配能力出色
- **用第三人称祈使句写作** — 将指令作为对 Agent 的直接命令

**打包确定性脚本**：
- **卸载脆弱/重复任务** — 给 Agent 经过测试的脚本来解析复杂数据集或查询数据库
- **优雅处理边缘情况** — 编写返回高度描述性、人类可读错误消息的脚本

### 验证指南

**发现验证**：测试 LLM 如何在隔离环境中解释描述，防止虚假触发

**逻辑验证**：确保分步指令是确定性的，不会迫使 Agent 猜测缺失步骤

**边缘情况测试**：强迫 LLM 寻找脆弱性、不支持的配置和 Web 工具固有的失败状态

**架构优化**：强制执行渐进式披露模式以减少 token 占用

### 项目亮点

1. **Agent 工程化** — 从"艺术"变成"工程"，建立标准化流程
2. **验证机制** — 用 LLM 验证 LLM 技能，形成质量控制闭环
3. **渐进式披露** — 解决 Agent 上下文膨胀的核心问题
4. **实战导向** — 来自 nanobot 等实际项目的经验总结

**随着 Agent 技能数量激增，缺乏标准会导致可发现性差、上下文膨胀、不可靠。skills-best-practices 为 Agent 生态提供了第一个标准化框架。**

DataClaw 代表了 AI 时代的"数据权利运动"：
